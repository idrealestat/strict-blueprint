## إعادة التوجيه: محتوى موجّه للوسيط العقاري فقط

نتخلّى عن مفهوم "شريط أخبار عام" ونحوّله إلى **مركز معلومات الوسيط العقاري** — كل ما يُعرض يجب أن يخدم قرار الوسيط اليومي.

## البنية الجديدة المقترحة

نستبدل `NewsBar` الحالي بثلاثة أشرطة مكدّسة (يمكن طيّها/فردها)، كلها تقرأ من نفس قاعدة بيانات مركزية تُغذّى عبر Edge Functions:

```text
┌─────────────────────────────────────────────────────────┐
│ 1) شريط المؤشرات الحيّة (Ticker مالي/عقاري)             │
├─────────────────────────────────────────────────────────┤
│ 2) شريط التنظيمي والرسمي (REGA / الإسكان / SAMA)        │
├─────────────────────────────────────────────────────────┤
│ 3) شريط أخبار السوق العقاري (مصادر متخصصة فقط)         │
└─────────────────────────────────────────────────────────┘
```

### الشريط 1 — مؤشرات الوسيط الحيّة `BrokerIndicatorsBar`
أرقام فقط، تتحرّك بسهم أخضر/أحمر. لا نصوص طويلة.

| المؤشر | المصدر | التحديث |
|---|---|---|
| SAIBOR 1M / 3M / 6M / 12M | SAMA | يومي |
| سعر الريبو الرسمي | SAMA | عند التغيير |
| متوسط هامش رهن عقاري (الراجحي/الأهلي/الرياض/البلاد) | جدول `bank_rates` يدوي + scraping أسبوعي | أسبوعي |
| مؤشر تداول العقاري TRESI | RSS تداول | يومي |
| متوسط سعر م² (الرياض/جدة/الدمام) | data.gov.sa (مرتبط مسبقاً بالمشروع) | شهري |
| أسعار الإسمنت/الحديد (مؤشر بناء) | RSS اقتصادي | أسبوعي |

### الشريط 2 — التنظيمي الرسمي `RegulatoryBar`
**هذا هو الأهم للوسيط** — كل ما يصدر عن الجهات الرسمية:

- **هيئة العقار العامة (REGA):**
  - تصاريح/قرارات/أنظمة جديدة من `rega.gov.sa/المركز-الإعلامي`.
  - تحديثات نظام الوساطة العقارية.
  - تحديثات رسوم الأراضي البيضاء.
- **وزارة الإسكان / سكني:** برامج جديدة، شروط دعم، إعلانات.
- **الهيئة العقارية / صندوق التنمية العقارية:** أسقف تمويل، شروط جديدة.
- **منصة إيجار:** تحديثات النظام، رسوم، إلزاميات.
- **هيئة الزكاة والضريبة:** ضريبة التصرفات العقارية، تحديثات VAT على العقار.
- **SAMA:** قرارات الفائدة، تعليمات الرهن العقاري للأفراد.

كل عنصر مع: تاريخ القرار، الجهة، شارة "إلزامي/اختياري"، رابط الوثيقة الرسمية PDF إن وُجد.

### الشريط 3 — أخبار السوق العقاري `MarketNewsBar`
RSS من مصادر **متخصصة فقط** (لا أخبار سياسية/رياضية):
- Argaam قسم العقارات.
- صحيفة مال — قسم عقارات.
- الاقتصادية — تصنيف عقارات.
- مدوّنة بيوت السعودية / عقار ماب (تقارير سوق).
- صحف محلية: قسم العقار فقط عبر فلترة كلمات.

**فلترة قسرية:** قبل الحفظ، نُسقط أي خبر لا يحتوي إحدى الكلمات: عقار، أرض، فيلا، شقة، إيجار، رهن، تمويل عقاري، تطوير، إسكان، فال، رخصة، REGA.

## قاعدة البيانات

```text
broker_indicators(code, name_ar, value, previous_value, unit,
                  change_pct, trend, source, source_url, updated_at)
  مثال code: SAIBOR_3M, REPO, TRESI, MORTGAGE_RAJHI, PSM_RIYADH

regulatory_updates(id, authority, title, summary, severity,
                   document_url, published_at, tags[])
  authority enum: REGA, MOH, SAMA, EJAR, ZATCA, REDF

market_news(id, title, summary, url, source, source_logo,
            category, published_at, trending)
  category: عقاري / تمويل / استثمار / تطوير
```

كل الجداول: SELECT للجميع (anon + authenticated)، الكتابة من service_role فقط.

## Edge Functions

1. **`fetch-broker-indicators`** (cron يومي 6 صباحاً):
   - يجلب SAIBOR/Repo من SAMA (HTML parse عبر deno-dom، أو Firecrawl إن توفّر).
   - يجلب TRESI من تداول.
   - يقرأ data.gov.sa لمتوسط أسعار م².
   - UPSERT في `broker_indicators` ويحسب `change_pct`.

2. **`fetch-regulatory-updates`** (cron كل 6 ساعات):
   - يفحص صفحات الأخبار الرسمية للجهات الست (REGA, MOH, SAMA, EJAR, ZATCA, REDF).
   - يستخرج العناوين الجديدة فقط (مقارنة بـ `published_at` الأحدث في الجدول).
   - يستخدم Lovable AI لتلخيص كل وثيقة في جملتين عربيتين + تصنيف severity (إلزامي/تنبيه/معلومة).

3. **`fetch-market-news`** (cron كل ساعة):
   - RSS من المصادر العقارية المتخصصة.
   - فلترة الكلمات الإلزامية.
   - UPSERT في `market_news` مع dedupe بالعنوان.

كل دالة تستخدم `EXTENSION pg_cron + pg_net` للجدولة (الإعدادات تُدخل عبر insert tool لأنها تحوي project URL).

## واجهة المستخدم

- 3 أشرطة مكدّسة في رأس الـ Dashboard.
- زر طيّ/فرد لكل شريط (حالته محفوظة في localStorage).
- بادج عدد التحديثات الجديدة منذ آخر زيارة على كل شريط.
- نقر على أي عنصر يفتح Drawer جانبي (RTL) فيه:
  - الملخّص الذكي.
  - رابط المصدر الرسمي.
  - زر "إضافة إلى ملاحظات CRM" و"مشاركة مع العميل عبر واتساب" (يستخدم Wasender الموجود).
- شريط التنظيمي يعرض شارة حمراء نابضة إذا فيه قرار "إلزامي" غير مقروء.

## ميزة إضافية: تنبيه ذكي للوسيط
عند صدور قرار REGA أو تغيّر فائدة SAMA، يُرسل إشعار داخل التطبيق (وعند تفعيل المستخدم: واتساب عبر Wasender) — لأن الوسيط يحتاج معرفة هذه الأمور فوراً، لا اكتشافها لاحقاً في الشريط.

## خطوات التنفيذ المقترحة

1. إنشاء الجداول الثلاثة + GRANT + RLS.
2. `fetch-broker-indicators` + cron يومي + بيانات بذرية (seed) للبنوك.
3. `fetch-regulatory-updates` + cron كل 6 ساعات + Lovable AI للتلخيص.
4. `fetch-market-news` + cron كل ساعة + قائمة فلترة.
5. مكوّنات الواجهة: `BrokerIndicatorsBar` + `RegulatoryBar` + `MarketNewsBar` + Drawer التفاصيل.
6. ربط زر "أضف لـ CRM" و"شارك واتساب".
7. حذف `useRealTimeNews` القديم و `NewsBar` القديم بعد التأكد.

## اختر مسار التنفيذ
- **(أ) كامل** — الأشرطة الثلاثة + Drawer + تنبيهات واتساب (أكبر قيمة، أطول وقت).
- **(ب) متدرّج موصى به** — نبدأ بالشريط التنظيمي (REGA/SAMA/EJAR) أولاً لأنه الأعلى قيمة للوسيط، ثم نضيف المؤشرات، ثم أخبار السوق.
- **(ج) مؤشرات أولاً** — `BrokerIndicatorsBar` فقط (أرقام SAIBOR/فائدة/أسعار م²) لأنه الأبسط وأكثر بصرية، ونحذف شريط الأخبار الحالي مؤقتاً.
